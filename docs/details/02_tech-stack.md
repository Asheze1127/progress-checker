# 技術スタック

---

## 0️⃣ 前提定義（最初に決める）

| 項目        | 内容                                                                 |
| --------- | ------------------------------------------------------------------ |
| プロダクト種別   | 社内ツール（ハッカソン運営向け）/ Webダッシュボード + Slack Bot 構成                       |
| 想定ユーザー規模  | ハッカソン参加者 + メンター + 運営 合計 数十〜70名程度 / 同時接続は数十人ほど / 将来拡張は基本不要        |
| 可用性目標     | イベント期間中に限定。99%程度で十分（短時間のダウンは許容）                              |
| セキュリティ要件  | Slack署名検証必須。GitHubトークン・LLM APIキーはサーバー側で管理（クライアント露出なし）AWSVPC+WAF     |
| パフォーマンス要件 | スループットはピーク数件/分程度で十分     |
| チーム体制     | hinata, tsubasaの二人体制 front: tsubasa(main)hinata(sub) back: hinata(main)tsubasa(sub)                          |
| リリース頻度    | 随時デプロイ                                           |
| 予算制約      | 不明一旦最低限で考えることとする          |
| 既存資産      | Slack ワークスペース（運営管理済み）/ GPTOSS120B |
